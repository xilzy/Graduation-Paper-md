# EXP-ABLATION-01：v3 最优架构的超参数测试与消融实验

- 日期：2026-07-02　协议同 `EVALUATION-metrics.md`；数据 `fusion_bench/`（irvis50 / medical48 / gfp30）vs 18 已发表方法。
- 模型：v3 = oc96-depth4 MoE（blend 决策图头, ws8, 12 路由专家+1 共享 top-2 稀疏, maxfuse 损失, ~4.11M, `models/Ours_v3_frozen`）。
- 目标：（1）列出最优架构中需要测试的超参数并给出结果；（2）针对每个创新点设计消融，量化其贡献，为论文提供依据。
- 口径：跳板机 8×H800。每个变体只相对 v3 改动**一个**因素，其余全同（20 epoch, bs10, patch170, weight `2 2 2 2 4 0 3`）。核心 12 指标：EN/MI/SD/SF/AG/SSIM/MS\_SSIM/Qabf/VIF/SCD/CC/Nabf。**v3 基线 = irvis 4 / medical 5 / gfp 5，合计 14。**

## 1. 最优架构中需要测试的超参数（总结）

| 类别 | 超参数 | v3 取值 | 说明/取舍 |
|---|---|---|---|
| MoE 规模 | 路由专家数 `n_routed` | **12** | 决定专家粒度/容量；下方 hp\_nr8 测 8 |
| MoE 规模 | top-k `k` | 2 | 每 token 激活专家数（稀疏度） |
| MoE 规模 | 共享专家数 `n_shared` | 1 | 常开专家，抗塌缩 |
| MoE 路由 | `routing` | softmax(+aux) | vs DeepSeek 无辅助损失（ab\_deepseek） |
| MoE 路由 | 任务条件 `task_cond` | True | 路由是否注入任务嵌入（ab\_notaskcond） |
| MoE 路由 | aux 权重 | 0.01 | 负载均衡损失系数 |
| 骨干 | 通道 `out_channel` | 96 | 已知 128 反降（v3 报告） |
| 骨干 | 深度 `depth` | **4** | vs 3（ab\_d3）；5 会 OOM |
| 骨干 | 窗口 `window_size` | 8 | vs 1 逐像素退化（ab\_ws1） |
| 融合头 | `fusion_head` | **blend** | vs direct（ab\_direct）/blenddetail/inn |
| 融合头 | `res_scale` | 0 | detail 残差强度（v3 关） |
| 损失 | `loss_mode` | **maxfuse** | vs orig（ab\_orig） |
| 损失 | 权重 `weight` | 2 2 2 2 4 0 3 | int/ssim/grad 分项权重 |
| 训练 | crops-per-task / lr / bs | 4000 / 1e-3 / 10 | 数据量与优化超参 |

其中 **n\_routed / depth / window\_size / fusion\_head / routing / task\_cond / loss\_mode** 是决定性因素，本文逐一消融；通道数已在 v3 报告中确定（96 最优）。
