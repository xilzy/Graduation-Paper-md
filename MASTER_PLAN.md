## 1. 课题目标与关键问题

将研究范围从单一 GFP–PC 显微融合，扩展到 **GFP–PC / 红外–可见光(IR-VIS) / 多聚焦(MFF)** 等多任务、多模态、多尺度互补的统一自适应融合框架，并结合算力资源引入 MoE 与分布式训练优化。

拟解决的 5 个关键问题：

1. **(通用 vs 专精)** 如何从特定医学显微融合扩展到综合多模态融合，平衡通用融合能力与任务特异能力。
2. **(多层次信息)** 如何在融合中同时保持局部细节、多尺度结构与全局语义，设计有效的多尺度与跨区域特征交互。
3. **(MoE 路由)** 如何设计适合融合任务的 MoE 路由，避免专家负载不均、专家冗余与任务间干扰，使容量扩展真正转化为融合质量。
4. **(无监督优化)** 如何在无真实融合标签下建立稳定有效、可跨任务的统一优化目标。
5. **(训练效率)** 如何在大模型 + 多任务条件下，通过分布式优化提升资源利用率与训练稳定性。

## 2. 现有基础：MDFNet 现状（起点）

**任务**：单一模态对 GFP–PC 融合，无 GT、无监督。

**网络**（`Networks/net.py: MODEL`）：
- 三分支多尺度结构，每支分别叠 1/2/3 个 ACM + 3 个 TM，三支输出相加 → 1×1 conv + tanh 出图。
- **ACM = 自适应卷积**（`Networks/layers.py: Conv2d`）：在普通卷积上加内容相关核生成，补全局语义。
- **TM = 窗口注意力 Transformer**（Swin 风格 `WindowAttention`/`TransformerBlock`）。
- 输入 `in_channel=2`（GFP-Y 与 PC concat），`out_channel=16`。

**损失**（`losses/__init__.py`）：
`L_total = L_content + α·L_structure`；`L_content = L_RMI + L_Intensity`；`L_structure = L_SSIM + L_Grad`。

**训练/评测**：10 epoch、bs64、Adam、单卡 4090；数据 `GFP-PCI_170.h5`（170×170 patch，5320 对）；评测为 `evaluation/` MATLAB 指标，对比 9 方法 + 损失项消融。

**影响扩展的关键代码事实（待重构）**：
1. `MODEL.forward` 三分支**共享同一 `self.conv` 与同一 `self.basicLayer`** —— "多尺度"实为同组权重的深度差异，非独立尺度参数。
2. 默认 `window_size=1` 且 `forward` 未真正使用 `patch_embed`；窗口为 1×1 时注意力退化为逐像素线性投影，长程建模未发挥。
3. `Train.py` 内层 `for index in range(batch_size)` **逐样本前向**，放弃 batch 并行，效率极低；数据管线写死 h5 / 写死 170 / 写死 `in_channel=2` / 单卡。
4. `intensity` 损失把融合图同时往两源均值拉，跨任务易冲突，需任务自适应。

## 3. 从 MDFNet 到目标的核心改造点

| 维度 | 现状 | 目标改造 | 关键问题 |
|---|---|---|---|
| 任务 | 单一 GFP–PC | 统一框架支持 GFP–PC / IR–VIS / MFF | (1) |
| 输入 | 写死 2 通道 | 任意模态对 + 模态/任务嵌入 | (1) |
| 容量 | 共享小网络 | TM 的 FFN → **MoE**（共享专家+路由专家）；可选 ACM 专家化 | (1)(3) |
| 多尺度 | 共享权重伪多尺度 | 独立尺度参数 + 真窗口注意力 | (2) |
| 损失 | 固定权重、面向 GFP–PC | 任务自适应、无 GT 的统一目标 + 负载均衡辅助损失 | (3)(4) |
| 训练 | 单卡、逐样本 | 真 batch + FSDP/专家并行 | (5) |

## 4. 分阶段实验计划

> 每个阶段对应论文可写的一节；每阶段内的每次改造都走 §5 的过程评测闭环并产出 EXP 记录。

### 阶段 0 — 基线复现与多任务底座
- 复现现有 GFP–PC 指标，作为对照锚点。
- 重构训练管线：去逐样本循环改真 batch；统一 `FusionDataset`（返回 `src_a, src_b, task_id, modality_meta`），接入 IR-VIS / MFF。
- `in_channel` 参数化；补回真窗口注意力；验证单任务不退化。
- **产出**：可多任务加载的统一基线，GFP–PC 不掉点。

### 阶段 1 — 单一稠密模型多任务联合训练（无 MoE 对照）
- 共享 backbone（+任务/模态嵌入）联合训练三任务。
- 实验：各任务单训 vs 联合训，量化**任务干扰**（任务干扰矩阵）。
- **产出**：任务干扰证据 + 通用/专精 baseline 数字（论证 MoE 必要性）。

### 阶段 2 — 引入 MoE（论文主创新，问题 (1)(3)）
- 插入点：`TransformerBlock` 的 `Mlp` → MoE-FFN（top-k 路由）；可选 ACM 专家化。
- 路由设计对比：路由条件（token / +任务 / +模态）；结构（纯路由 vs **共享专家+路由专家**）；负载均衡（aux loss、容量因子）。
- 记录专家负载分布、专家–任务激活热力图、利用率/冗余。
- 扩展性：专家数 {4,8,16,…}×top-k → 质量–参数量–FLOPs 曲线。

### 阶段 3 — 无 GT 的统一多任务损失（问题 (4)）
- 将 SSIM/RMI/梯度/强度损失推广为**任务自适应**（IR 保热目标显著性、MFF 保清晰区、GFP–PC 保功能信息+对比度）。
- 实验：固定权重 vs 任务条件权重 vs 不确定性自动加权（learnable σ）；加 aux loss 后优化稳定性对比。

### 阶段 4 — 分布式训练与效率（问题 (5)）
- 在现有 FSDP/torchtitan 设施上做 MoE+Transformer 分布式：FSDP2 分片 + 专家并行(EP) + 激活检查点 + bf16。
- 实验：单卡 vs FSDP8(±EP) 的 MFU/TPS/显存/扩展效率（复用已有 A3B MoE MFU/TPS sweep 方法学）。

### 阶段 5 — 综合对比 + 消融（成文）
- SOTA 对比（各任务代表方法）、逐项消融（真窗口注意力 / 独立多尺度 / MoE / 共享专家 / 任务自适应损失 / 负载均衡）、跨任务泛化（zero-shot/少样本）。

## 5. 过程性评价方法（核心）

### 5.1 三个支点（无 GT 下确认"对/有优势"）
1. **每次改造都带固定基线对照**（不是只在阶段五）：基线方法融合输出预跑存盘，S1 起即一起算分。
2. **分源拆解诊断**：`MI(F,A) vs MI(F,B)`、`SSIM(F,A) vs SSIM(F,B)`、Qabf 对 A/B 的边缘转移率；其**平衡度**暴露模态偏置/任务干扰。
3. **任务适配面板 + 退化检测**：不同任务不同关注点；并设退化检测器（全黑/复制单源/伪影/NaN）兜底。

### 5.2 四级"测试结果阶段"
| 阶段 | 何时 | 数据 | 指标 | 判据 |
|---|---|---|---|---|
| **S0 冒烟** | 每次代码改动/启动后 | 1 batch / 3–5 对 | 退化检测 + 肉眼出图 | 非全黑、非复制单源、无 NaN |
| **S1 迭代评测**（核心） | **每次算法改造后** | 冻结 probe 子集（各任务 10–15 对） | 核心 9 项 + 诊断 + 固定基线 | 平均排名↑且不输基线；定位回归 |
| **S2 里程碑** | 每阶段收尾 | 完整测试集 | 核心面板 + 全基线 | go/no-go |
| **S3 终评** | 成文前 | 全测试集×全任务 | 全量 + 全基线 + 显著性 | 论文定论 |

### 5.3 指标选取
**核心 9 项（恒报）**：EN、MI（信息量）；SD、SF、AG（图像统计）；SSIM、MS-SSIM（结构）；Qabf（梯度转移）；VIF/VIFF（人眼保真）。
**诊断项（S1 定位问题）**：Nabf（伪影↓）、SCD（互补保留↑）、Qcv↓/Qcb↑、分源拆解比值、CC（已知短板监控）。
覆盖全部指标族，保证全面、可发表、避免 cherry-pick；所选项多为我们设计目标天然占优者。

### 5.4 聚合总分：**平均排名（rank-based）**（已拍板）
- {我们 + 固定基线} 在 9 项核心指标各自排名（↓类指标 Qcv/Nabf 反向），取**平均排名**为单一总分。
- 对量纲鲁棒、最抗刷分；"平均排名 < 1.x" 等价于"在多数指标上赢过基线"。
- 配套永远同看：指标族雷达图 + 分源平衡条 + 逐图折线（总分看趋势，分解定位问题）。

### 5.5 图片子集
- **probe（S1，冻结）**：各任务 10–15 对，固定 seed，覆盖难度梯度（含难例：GFP-PC 噪声对 #10–15、IR-VIS 低光/烟雾、MFF 大散焦）。
- **完整测试集（S2/S3）**：GFP-PC 30 对（沿用原划分）、IR-VIS（TNO/MSRS/RoadScene）、MFF（Lytro/MFFW）。
- 基线方法输出预跑存盘，S1 起对照。

### 5.6 迭代闭环
改造 → S1 评测 → ①看平均排名（有效性）②看哪族/哪源/哪类图回归（问题诊断）→ 针对问题再改 → 重复，直到平均排名领先且各族均衡。
