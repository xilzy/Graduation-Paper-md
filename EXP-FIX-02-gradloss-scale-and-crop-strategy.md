# EXP-FIX-02：梯度损失尺度无关化 + 裁剪策略抉择（问题 1）

- 日期：2026-06-24　关联代码：`losses/__init__.py: joint_grad`、`mm_fusion_dataset.py`
- 结论：① 把 `joint_grad` 的 `/(170*170)` 改为 **`/(H*W)`（=空间均值，尺度无关）**——170 完全等价、其它尺寸正确。② 裁剪策略选 **统一裁剪尺度（不按数据集各异）**，理由是多任务混批要求批内同尺寸；修复让"裁剪尺寸"从被 170 锁死变成可自由选的超参（具体尺寸在 EXP-FIX-03/问题2 定）。

## 1. 问题由什么引发、会导致什么、和什么关联
原 `joint_grad` 把梯度残差在空间维**求和**后除以**写死的 `170²`**。这个常数与输入实际尺寸解耦：一旦不是 170×170，求和项数变了、分母没变 → **梯度损失被按 `(H*W)/170²` 倍放大**，且每个任务因尺寸不同放大倍数不同 → 多任务联合训练时各任务"有效损失配比"被尺寸悄悄改变，训练被污染、不可比、不可复现（隐蔽 bug，比报错更危险）。
- 关联：与"裁剪策略"直接耦合（正是它逼着我们统一裁 170）；与 ACM 方形假设（问题4）同源于"统一方形 patch"决策；强度损失的跨任务冲突是另一条线（任务自适应损失）。

## 2. 解决方法与原理
```python
# 原: Grad_value = L1(|grad_fuse| - joint_grad) / (170*170)
H, W = fused.shape[2], fused.shape[3]
Grad_value = L1(|grad_fuse| - joint_grad) / (H * W)   # 求和÷像素数 = 均值，尺度无关
```
对 170×170：`/170²` 与 `/(H*W)` 完全相同 → 对已发布 MDFNet 与既有 GFP-PC 结果**零影响**；对任意尺寸：重新等于"梯度残差均值"，与 SSIM/RMI 同处一个量纲。

## 3. 探索与验证（实验数据）
**(a) 量化原 bug 的尺度放大**（同一随机批，OLD vs NEW）：

| 尺寸 | OLD `/170²` | NEW `/(H*W)` | OLD/NEW |
|---|---|---|---|
| 170² | 0.1922 | 0.1922 | 1.00× |
| 256² | 0.4328 | 0.1909 | 2.27× |
| 358² | 0.8486 | 0.1913 | 4.43× |
| 480×640 | 2.0316 | 0.1911 | **10.63×** |

→ NEW 在所有尺寸 ≈0.191 恒定（尺度无关）；OLD 随面积线性膨胀。

**(b) 修复后非 170 尺寸能否正常训练**（三任务，2 epoch，crops_per_task 1500）：

| patch | structure(ep1→ep2) | content | s/epoch |
|---|---|---|---|
| 170 | 3.74 → 1.58 | 2.65→1.92 | ~60s |
| 256 | 3.86 → 1.89 | 3.41→2.74 | ~130s |

→ 256 的 **structure（含梯度项）与 170 同量级**（不再是 2.3× 膨胀），均正常收敛；256 计算约 2.2×（像素更多）。

**(c) 能否"每个数据集不同裁剪尺寸"** —— 直接 `torch.stack([(1,170,170),(1,256,256)])` **报错**（批内张量必须同尺寸）。

## 4. 抉择：统一裁剪尺度 vs 各数据集各异
- "各数据集不同裁剪尺寸"在**数值上**已被(a)(b)解禁（损失尺度无关），但**工程上**(c)表明：要混入同一 batch 必须同尺寸。各异尺寸只能走**单任务批**（每个 batch 只一个任务/一个尺寸）或 padding。
- 单任务批会丢掉"每个 batch 同时看到三任务"的梯度混合稳定性（对联合训练、抑制任务漂移很重要），且实现更复杂。
- **故选统一裁剪尺度**：保留混批稳定性、实现简单、损失可比。修复的真正价值 = 把"裁剪尺寸"从被 170 锁死，变成一个**可按上下文/显存/窗口注意力整除性自由选择的超参**。

## 5. 结论 / 下一步
- `joint_grad` 尺度无关化（已改、已验证、170 等价）。
- 裁剪策略 = **统一尺寸**；具体取多大（170 vs 224 vs 256…）与 `crops_per_task` 一起在 EXP-FIX-03（问题2）按"收敛+退化+显存/速度"权衡选定。
