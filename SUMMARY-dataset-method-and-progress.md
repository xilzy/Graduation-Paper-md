## 1. 数据集创建方法

把三类异构源（GFP-PC 显微 / MSRS 红外-可见光 / Harvard AANLIB 脑医学）统一成同一网络契约的训练流，四条核心：

1. **统一到亮度（Y）域**：彩色源（GFP/VIS/PET/SPECT）→ BT.601 YCbCr 只取 Y；灰度源（PC/IR/MRI）即 Y。两路 Y∈[0,1] concat 成 `(2,H,W)`，沿用 `in_channel=2`，网络通道不动。颜色 CbCr 不进训练，**推理时回填**（`ycbcr.py`）逆变换出 RGB。
2. **双配对 schema**（`mm_fusion_data.py`，向后兼容）：`suffix`（GFP-PC 同目录不同后缀）+ `folder`（MSRS/医学 分目录同名 + 显式 train/test）。
3. **统一 170×170 方裁、不缩放**（`mm_fusion_dataset.py`）：尺寸不一（358/640×480/256）一律裁切，小图反射 padding，B 仅在尺寸不一致时双线性对齐 A。
4. **按任务配额平衡**（`crops_per_task`，固定 seed）：各任务贡献相近裁块（实测 GFP 3894 / IR-VIS 3249 / 医学 3468），防大集淹没小集。

数据落地：医学下载 `Havard-Medical-Image-Fusion-Datasets`（810 对 256×256）→ `data/Harvard-Medical`（train 578/test 48，A=PET/SPECT 伪彩，B=MRI 灰度）；MSRS、GFP-PC 沿用。

## 2. 设计原因 / 原理

- **统一 Y 域 + 色度回填**：IR-VIS/医学/显微三域**社区共识**（PIAFusion/SwinFusion/GeSeNet 源码 `RGB2YCrCb`→融Y→拼CbCr；Dif-Fusion/MDC-RHT 论文明述）。结构在亮度（与灰度源重叠、需融合），颜色在色度（灰度源无对应、原样保留）。→ 三任务共用一个网络契约，不必改通道。
- **裁不缩放**：梯度损失 `joint_grad` 归一化写死 `/170²`，缩放会扭曲亮度/梯度统计 → 统一 170 方裁。
- **任务配额平衡**：数据量悬殊（1083 vs 118）不平衡会放大任务干扰。
- **[0,1] 归一**：主流实现一致（仅 MATR 异类）。
- **免配准**：三套数据均已预配准。
- 终极目标：**一套权重对三类图像都融合好**，而非每类单训一个模型。

## 3. 做了哪些测试 + 结果

- **S0 耦合冒烟**：2/3 任务数据集构建成功（裁块均衡、(1,170,170)、Y∈[0,1]）；稠密与 MoE 训练前向/反向/存档**全程无报错**。
- **稠密 8-epoch 训练**：loss 7.15→**3.92** 单调收敛（165s/ep）。
- **MoE 8-epoch 训练**：loss 11.38→**9.46**，aux 稳定 ~9.12（209s/ep）。
- **退化自检**（各任务 probe 5 图）：

|---|---|---|---|---|
| 稠密 ep8 | PASS | **FAIL 近黑(0.008)** | PASS | SOME FAIL |
| MoE ep8 | PASS | **PASS(0.095)** | PASS | ALL PASS |

## 4. 尝试的行为 + 结论

- **尝试①：原网络 + 原始损失训统一数据集** → 数据管线 OK（GFP/医学正常收敛），但 **IR-VIS 塌成近黑**（白天亮可见光也 0.003–0.007）。根因：GFP-PC 标定的对称损失（SSIM 对 B 权重 5×、强度拉两源均值）把输出往暗红外（IR 均值~0.06）拽。**结论：复现了跨任务干扰，是 MoE/任务自适应必要性的直接证据。**
- **尝试②：改网络+损失（新建 `net_moe.py`/`train_moe.py`，不动原网络）** —— 共享专家+top-k 路由 MoE-FFN、任务条件路由、负载均衡 aux、任务自适应 max-强度损失。→ 收敛、S0 三任务全过、**IR-VIS 塌黑解除**。
- **诚实边界**：本轮同改架构+损失，未拆贡献；MoE 逐源 Y-corr 偏低，S0 判不了质量 → **质量须走 S1 三轴平均排名**。
- **附带发现**：ACM（`layers.py`）假设方形输入，非方形整图推理会崩（训练 170 方裁不受影响，自检已方裁兜底）。
