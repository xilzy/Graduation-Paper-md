# EXP-FIX-01：修复 ACM 自适应卷积的"方形输入"假设（问题 4）

- 日期：2026-06-24　关联代码：`Networks/layers.py: Conv2d.forward`；验证 `mm_infer_check.py`
- 结论：**一行根治（方案 1）成立**——用卷积算术推导真实输出尺寸替代 `int(np.sqrt(l))`，非方形整图推理从崩溃→正常，方形输入数值完全等价。

## 1. 问题是什么 / 由什么引发
ACM（内容自适应卷积）forward 末尾把 `unfold→matmul` 得到的展平结果 reshape 回 2D：
```python
return torch.matmul(out, x_un).view(b, self.oc, int(np.sqrt(l)), int(np.sqrt(l)))
```
`l` 是滑窗个数（= 输出像素数 H_out×W_out）。作者用 `int(sqrt(l))` 同时当 H 和 W，**硬性假设 H_out==W_out（方形）**，且没把真实 (h,w) 传下来。

## 2. 会影响什么
- 方形输入（GFP 358²、医学 256²、训练裁块 170²）：`sqrt(l)`=边长，恰好对，所以一直没暴露。
- 非方形（MSRS 640×480）：`l=307200`，`int(sqrt)=554`，`view(...,554,554)` 需 306916≠307200 → **RuntimeError 直接崩**。
- 影响面：**训练不受影响**（只喂 170 方裁块）；崩的是**非方形图的整图推理/评测**——即 IR-VIS 最终成图、S1/S2 评测过不去。

## 3. 探索过的方案与取舍
| 方案 | 做法 | 取舍 | 采纳 |
|---|---|---|---|
| 1 一行根治 | 用 conv 算术 `ho=(h+2p−d(k−1)−1)//s+1`（w 同）推导真实输出尺寸 | 改 1 处、方形完全等价、不影响已训权重、支持任意尺寸 | ✅ |
| 2 推理前中心方裁 | 裁到 min(H,W) | 丢边缘、评测不完整，仅够 S0 自检 | ⛔ 退役（原 mm_infer_check 兜底，现已移除） |
| 3 整图分块推理 | 重叠 patch 推理拼回 | 不改网络但有拼接缝、实现复杂 | 备选 |
| 4 pad 到方形 | pad 后裁回 | 边界伪影 | ⛔ |

选方案 1：最小改动、零副作用、彻底。

## 4. 实施与验证
改动（`layers.py`）：
```python
ho = (h + 2*self.padding[0] - self.dilation[0]*(self.ks-1) - 1)//self.stride[0] + 1
wo = (w + 2*self.padding[1] - self.dilation[1]*(self.ks-1) - 1)//self.stride[1] + 1
return torch.matmul(out, x_un).view(b, self.oc, ho, wo)
```
验证：
- 多尺寸前向 480×640 / 256² / 170² / 358² / 300×500 **全部 OK，输出尺寸 == 输入尺寸**（之前 480×640 崩）。
- 方形输入：matmul 计算完全不变，只是 reshape 解释正确 → 数值等价，不影响已训模型。
- 端到端：用稠密 model_8 对 MSRS **整图 640×480** 推理成功，输出存为原生 640×480（不再需要方裁）。
- 附带确认：整图下 IR-VIS 仍 BLACK（mean 0.008）——**证明塌黑是损失/训练问题，不是裁切假象**（与 EXP-0-06 分析一致），下一步靠任务自适应损失/MoE 解决。

## 5. 关联
- 与"统一 170 方裁"（问题 1）同源：正因训练用方裁块才一直没暴露此假设；本修复让整图彩色推理/评测得以打通，是 S1 评测的前置。
