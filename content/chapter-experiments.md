## 4.0　本章导读（写作要点）

- 一句话主线：把已中稿的**单任务 GFP–PC 显微融合（MDFNet）**扩展为**统一多模态融合框架**（IR-VIS / 医学 / GFP-PC 三任务一套权重），并以 **MoE + 分布式训练**为核心创新；实验证明 Ours（v3）在三任务上综合进入 SOTA 前列、逐创新点消融均正向、训练效率显著提升。
- 与会议论文的差异（需在导读中交代）：① 任务从单一 GFP–PC 扩到三模态统一；② 对比方法从 9 个扩到 **18 个**近年代表方法；③ 指标从 9 项扩到**核心 9 + 诊断 + 平衡感知 + 功能轴**，聚合改为**平均排名**（抗刷分）；④ 新增 **MoE / 决策图头 / 任务自适应损失**的消融与**分布式训练效率**实验。
- 全章结论锚点（反复回指）：**Ours 拿下 7 个全场第一（MI×3、VIF×3、gfp:Qabf）**；Top-3 三模态 4/5/5、Top-5 三模态 7/6/5（均 ≥5）；综合平均排名稳居每类前 4–5。

## 4.1　实验设置（Setup）

### 4.1.1　实现细节与实验环境
写作要点：
- 框架：PyTorch；训练/测试同一实现。
- 骨干：三分支多尺度 = 自适应卷积模块(ACM) + 窗口注意力 Transformer 模块(TM)；TM 的 FFN 替换为 **MoE-FFN**（共享专家 + top-k 路由专家）。输入统一 `in_channel=2`（两路亮度 Y concat），`out_channel=96`，depth=4，window_size=8。
- 训练超参：patch 170×170、bs=10、20 epoch、Adam、lr=1e-3；损失权重 `int/ssim/grad = 2 2 2 2 4 0 3`（maxfuse 模式）；负载均衡 aux 权重 0.01；每任务裁块配额 crops_per_task≈4000。
- 硬件：跳板机 8×H800（GPU 集群）；分布式训练见 §4.4。
- 【表 4-1】实现与训练配置汇总（模块数/通道/深度/窗口/专家数/top-k/epoch/bs/lr/优化器/硬件）。

### 4.1.2　数据集
写作要点（详见 `../DATASETS_AND_PREPROCESSING.md`、`../EXP-0-06-unified-multitask-dataset.md`）：
- 三任务三数据集：**IR-VIS = MSRS**（train 1083 / test 361，640×480，抽样测试 50）、**医学 = Harvard AANLIB**（PET/SPECT–MRI，train 578 / test 48，256×256）、**显微 = GFP-PC**（John Innes，train 118 / test 30，358×358）。
- 统一预处理契约（社区共识「fuse-Y / recombine-CbCr」）：彩色源→YCbCr 只取亮度 Y，灰度源即 Y；两路 Y∈[0,1] concat 成 (2,H,W)；色度 CbCr 不进训练，**推理时回填**逆变换出 RGB。
- 统一 **170×170 方裁不缩放**（因梯度损失归一化写死 /170²；小图反射 padding；B 仅在尺寸不一致时对齐 A）。
- **按任务配额平衡**裁块（固定 seed），防大数据集（MSRS）淹没小数据集（GFP 118 对）。
- 【表 4-2】三数据集规格（任务/来源/train-test 对数/分辨率/模态 A(彩色·功能)/模态 B(灰度·结构)/测试评测张数）。

### 4.1.3　评测指标与聚合方法
写作要点（详见 `../EVALUATION-metrics.md`）：
- **核心 9 项**：EN、MI、SD、SF、AG、SSIM、MS_SSIM、Qabf、VIF（均↑，越大越好）。
- **诊断项**：Nabf（伪影，↓）、SCD（互补保留，↑）、CC（与源相关，↑）；分源拆解 MI_A/MI_B、SSIM_A/SSIM_B。
- **平衡感知 *_hm**（逐源调和平均 soft-min，抗「复制单源」刷分）：SSIM_hm/MS_SSIM_hm/Qabf_hm/VIF_hm/MI_hm；两轴分开排名（FIDELITY 轴杀「复制/模糊」，QUALITY 轴杀「全图模糊」）。
- **功能轴**：FuncCorr / FuncSal（只在功能源显著区度量，检验稀疏功能信号是否被注入）。
- **彩色任务 RGB 计分协议**：medical/gfp_pc 融合 Y 与源 CbCr 重组逆变换成 RGB 后 `rgb2gray` 计分（对齐仓库 `infer_fusion.py` 与 MATLAB `main.m`）；irvis 保持灰度。
- **正确性保障**（集群无 MATLAB）：公式逐项对齐 `evaluation/*.m` + 单元性质测试（自融合 SSIM=1、常数图熵=0）+ 平凡 Avg/Max 去欺骗自检。
- **聚合 = 平均排名（rank-based）**：全部方法在核心指标上各自排名取均值，越低越好；对量纲鲁棒、最抗刷分。
- 【表 4-3】指标定义/方向/含义速查（核心 9 + 诊断）。

### 4.1.4　对比方法
写作要点（详见 `../SUMMARY-comparison-18methods.md`、`../EXP-CMP-01..14`）：
- 在**统一标准化基准**上复现 **18 个**代表方法，覆盖：Transformer 分解（CDDFuse、SwinFusion、DATFuse）、通用/自编码 CNN（IFCNN、DenseFuse、U2Fusion、RFN-Nest、LRRNet）、语义/光照/目标感知（SeAFusion、PIAFusion、TarDAL）、配准+融合（MURF）、扩散（DDFM）、传统方法（LP、DWT、DTCWT、GTF、NSCT*）。

- 关键复现可靠性注记：U2Fusion 作者 TF1 权重逐层移植（对拍误差 3.6e-7）、MURF 经 nvidia-tensorflow 跑通 H800、DDFM 校验完整 2.21GB 扩散权重。

## 4.3　消融实验（Ablation study）

### 4.3.1　创新点消融
写作要点（详见 `../EXP-ABLATION-01-v3-innovations.md`、`../EXP-ABLATION-PARAM-v3.md`）：
- 以完整 v3 为基线，每次**完全去掉一个创新点**重训 20 epoch，在同一张 18 方法大表里重新排名，统计三模态 Top-3 个数变化（判优看 v3 的「优势指标保持度」）。
- 结论一句话：**去掉任一创新都掉点，v3 是各因素的联合最优**；重要性排序 **决策图头 ≫ maxfuse ≈ 窗口注意力 > 专家数/深度/路由/任务条件 > MoE 边际项**。
- 决策图头是最决定性卖点：去掉后三模态全面崩塌（Top-3 合计 14→4，−10），因为 `F=w·A+(1−w)·B` 把融合图锚定为两源凸组合，无条件继承动态范围（SD/EN）并线性保留双源互信息（MI/VIF/SCD）；直接回归缺此约束被拉向低对比度均值图。
- MoE 的定位（诚实叙述）：早期「MoE 路由不如共享专家」的负结果，在**决策图头 + 真实注意力 + 细粒度专家（12）**的组合下被扭转——MoE 现为 irvis 净 +1 且不牺牲其他模态；贡献为边际但方向明确为正，与专家数强联动。

## 4.4　训练效率与分布式优化（扩展贡献，问题(5)）

写作要点（详见 `../EXP-INFRA-01-training-speedup.md`、`../EXP-INFRA-02-sdpa-grouped-moe-ddp.md`、`../EXP-INFRA-03-grouped-moe-ddp-evidence.md`）：
- **profile 先行**：v3 单步 CUDA 时间瓶颈是 **MoE 稀疏专家调度**（大量小 linear ~45% + 稀疏 top-k 的 index/copy ~18%+7% + 注意力 bmm 11.6%），而非算力——决定优化取舍。
- 逐项优化与实测（why + effect）：
  - **torch.compile** 图融合降 kernel launch：560→472 ms/step（1.19×）✅
  - **bf16 autocast 反例**：560→1540 ms（2.7× 变慢）——小算子/index/cholesky 主导的负载，bf16 cast 开销 + 小 GEMM 用不满张量核，得不偿失 ❌
  - **稠密 batched-MoE 反例**：此 token 量（~29 万/前向）下 OOM ❌；稀疏 + compile 更实际。
  - **DDP-4 数据并行 + compile（历史端到端）**：epoch 594s→**197s（≈3.0×）** ✅；该口径混有 DataLoader/编译等开销，不能直接视为纯通信效率。
  - **续作（INFRA-02/03）**：SDPA 省约 7.5GB；**容量受限分组 MoE**把 E 个小 linear 收敛为 2 次批量 GEMM，和 compile 联合达到 1.29×；E=12→32 的单步降幅由 9.2% 扩大到 36.2%；α=1.25 的冻结探针丢弃 0.80%、输出 MAE 3.67e-5。
  - **分布式复核（INFRA-03）**：8 MiB 两桶 + bucket-view + fused Adam + static graph；1→8 卡扩展效率 **99.08%**，通信模式差异低于步内噪声；单 rank 2/5/10ms 停顿几乎 1:1 传导为全局损失。
- AI-infra 洞见（可直接写入论文）：① profile 决定取舍；② bf16 非万灵药；③ grouped 的收益来自“规则执行形状×compile”，并随专家数增长；④ 当前 4M 模型不是 NCCL 受限，分布式首要风险是输入/计算 straggler，FSDP/EP 暂不匹配。
  - 【表 4-15~4-25 + 原理/证据图】`已就绪于 content/section-efficiency.md 与 INFRA-03`。

## 4.5　讨论、局限与本章小结

写作要点：
- **达成**：一套权重三任务综合进入 SOTA 前列（平均排名 irvis#5 / medical#4 / gfp#4），7 个全场第一，逐创新点消融全正向，训练 3.0× 提速；从中游（11–14 名）跃升。
- **局限（诚实）**：① 全场第一集中在信息/保真族（决策图凸组合天然强项），细节/锐度族（SF/SD/EN）仍非第一，被 CDDFuse/DenseFuse/SwinFusion 压住；② irvis 第 5 个 Top-3 卡在极紧 SD 墙（41.0 vs #3 的 41.4），加宽/加深/IR-VIS 专精/INN 高频分支充分实验（~20 组）均未翻越，锐化-平滑 Pareto 互斥——已确认为当前架构的 Pareto 上限；③ 评测规模 30–50 图/单 seed，终评需全测试集 + 多 seed + 显著性检验。
- **未来工作**：分解-重建强骨干（INN 高频分支 + 相关性分解损失 + 两阶段）把 SD 推过 41.4；完成 grouped 不同 α 的独立训练与多 seed 显著性终评；仅在跨节点、参数或专家规模显著扩大后重新评估 EP/FSDP。
