# EXP-0-04：加 LR 衰减重训（修复过锐化 / 重构管线验证）

- 日期：2026-06-23
- 所属阶段：阶段 0（deliverable 0.10：验证单任务不退化）
- 结果级别：S1（probe 15，RGB→rgb2gray 口径）
- 关联代码：`train_fusion.py`（加 `--lr-decay`，StepLR step=1 gamma=0.8）
- 证据：`logs/p0_gfp_pc_lrd/log.csv`、`reports/gfp_pc_lrd_probe/`、权重 `models/p0_gfp_pc_lrd/`

## 1. 改造内容
按论文"学习率衰减 0.8"加 StepLR（每 epoch ×0.8）。其余同 EXP-0-01 重构管线（真 batch + FusionDataset，window=1，α=2）。

## 2. 结果
LR 1e-3→1.34e-4，loss 平滑收敛 8.36→3.65（旧无衰减止于 4.12）。

probe（vs 官方 MDFNet 与旧重训）：

## 3. 分析
1. **过锐化/伪影回归已修复**：Nabf 0.487→0.111（与官方 0.106 相当）；SF/AG 大幅回落。LR 衰减让模型收敛到更平滑的解。✓
2. **重构管线验证通过**：能稳定收敛、无伪影、产出非退化且模态平衡(Balance 0.582)的融合图——管线本身正确。
3. **但未达"与官方逐指标 parity"**：新模型是**另一个工作点**——保真更高/更平衡(SSIM_hm、MI_hm、Balance 全面高于官方)，但锐度/EN 更低(偏平滑)。官方 MDFNet 走的是"高锐度高对比、低保真"的增强取向。
   - 原因：官方的 h5 训练数据、随机划分、确切 LR 步骤/权重/选取的 checkpoint 我都没有,无法逐位复刻;LR 衰减把解推向了更忠实两源的方向。
4. **质量轴自检仍 FAIL**：通用指标测不到 GFP-PC 要点的老问题(EXP-0-02)，待 #7 功能指标解决。

## 4. 决策
- **不强求逐指标复刻官方 checkpoint**（无原始 recipe，且意义不大）。**官方 `models/10` 作为冻结参照点保留**;**新重训 `p0_gfp_pc_lrd` 作为我们后续可训练的 baseline**。
- deliverable 0.10 视为达成(管线验证 + 修复回归);parity 的严格含义改记为"管线正确、无退化、产出合理"。
